目的
--
本模板用于教授学员如何编写 `Instructions` 文件，说明应当如何引导智能助理在特定工作场景中表现。内容通俗、结构清晰，既可用于开发场景，也可用于非软件（如写作、研究、行政流程）任务训练。

何时使用
--
- 团队希望设定统一风格或流程时。
- 需要把常见约定、禁忌、检查点和优先级告诉助理时。

模板结构说明（必备部分）
--
1. 头部（YAML frontmatter）
   - `title`: 文件标题
   - `scope`: `workspace` 或 `file`（决定是否全局生效）
   - `description`: 一句概述，包含触发关键词（便于发现）

2. 目标（Purpose）
   - 简明写出本说明希望达到的目标（1-2 行）。

3. 适用范围（Scope & When to use）
   - 列举本说明适用的文件类型、任务或场景（例如：撰写 PR 描述、生成周报、校对合同）。

4. 必要规则（Hard rules / Musts）
   - 强制执行的要求（例如：禁止公开密钥、必须包含审查人、格式化日期为 YYYY-MM-DD）。

5. 推荐做法（Preferences / Shoulds）
   - 风格偏好与建议（例如：用被动语态还是主动语态；每段不超过 5 行）。

6. 禁止项（Don'ts）
   - 明确列出助理不得执行或返回的内容类型。

7. 示例与演示（Examples）
   - 提供 2-3 个具体示例（输入 → 期望输出），覆盖常见用例。

8. 验证与检查清单（How to validate）
   - 训练或使用时如何判断输出是否合格（打分标准或核对项）。

9. 迭代说明（How to iterate）
   - 当发现歧义或需更新时，如何修改本文件并通知团队（提交 PR、标注变更记录）。

10. 非软件场景示例（可直接复制使用）
   - 场景 A：写作助手（生成培训材料）
     - 目标：输出面向初学者的 800 字教学稿，包含示例与练习题。
     - 必须：语言简明，避免行业术语；最后给出 3 个练习题及参考答案。
   - 场景 B：行政流程（撰写会议纪要）
     - 目标：把录音要点整理为要点列表，并标注待办与责任人。
     - 必须：每条待办包含截止日期与负责人。

示例（完整小样）
--
---
title: "撰写周报指南"
scope: "file"
description: "周报, 每周, 项目进展, 待办"
---

目标：为团队成员生成结构统一、重点明确的周报节。本周报面向跨职能团队，篇幅控制在 200-300 字。

必需项：
- 开头一句本周摘要（1 行）。
- 三个小节：完成项、进行中、下周计划。每项最多 2 行。

示例输入："本周在 X 功能完成了 A 测试，遇到 B 问题，计划下周部署到测试环境。"

期望输出：见上面规则格式化后的周报文本。

如何使用此模板（快速指南）
--
1. 复制本模板为项目专用文件，保存到团队约定目录（例如 `.github/instructions/` 或 `templates/`）。
2. 在 `description` 中放入可搜索关键词（便于助理加载）。
3. 提供 2-3 个高质量示例，覆盖正常与异常场景。
4. 定期（比如每季度）回顾并更新“必须项”和“示例”。

训练与教学建议
--
- 在课堂演示中，先展示“示例输入→示例输出”，让学员观察差异，再展示 YAML 头部如何影响加载。
- 练习题可让学员修改模板以支持新场景（例如把“周报”改成“月度报告”）。

常见注意事项
--
- 保持 `description` 简短且包含触发词。
- YAML 中避免使用制表符（tabs），使用空格并校验语法。
- 若规则可能影响隐私或合规，明确标注并联系法律/合规负责人。

反馈与变更记录
--
- 在文件底部添加“变更记录”段，记录修改日期、修改人、简要说明。

结束语
--
此模板旨在降低上手成本，帮助学员与团队快速创建有效的 `Instructions` 文件。需要我把此模板另存为不同版本（精简版 / 教学版 / 合规版）吗？

字段说明
--
- name
   - 类型: 字符串 (String)
   - 描述: 指令集的名称。用于在聊天界面或配置列表中识别该指令集。
   - 示例: "workspace-instructions", "Code Review Guidelines"

- description
   - 类型: 字符串 (String)
   - 描述: 对指令集内容的简要描述。当用户在聊天界面中选择或悬停在指令集上时，通常会显示此描述，帮助用户理解其用途。
   - 示例: "工作区指南：包含项目目标、常见命令及代码约定。"

- applyTo
   - 类型: 字符串 (String) (通常遵循 Glob 模式)
   - 描述: 指定这些指令适用于哪些文件。可以使用通配符来匹配文件路径。
   - 示例:
      - "**/*" (适用于所有文件)
      - "src/**/*.ts" (仅适用于 src 目录下的 TypeScript 文件)
      - "*.md" (仅适用于 Markdown 文件)
