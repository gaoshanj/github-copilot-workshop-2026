# Plan Mode Demo: E-commerce Order System

This directory contains a requirement document for a complex feature.
**Goal**: Demonstrate how **Plan Mode** helps decompose a large requirement into actionable steps.

## Scenario
We need to implement a new "Order Processing" module for an e-commerce system.

## Task for Copilot
1.  Open `requirements.md`.
2.  Type `@workspace /plan Implement the order processing system based on requirements.md`.
3.  Observe how Copilot breaks down the task into:
    -   Database Schema Design
    -   API Endpoint Definition
    -   Service Layer Logic
    -   Integration Tests

## Files
-   `requirements.md`: The input requirement document.
